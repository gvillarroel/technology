# Seed Requirements

## Objective

Provide the smallest practical Astro seed that builds a public GitHub Pages site from governed data kept
outside the repository. A private GCS manifest defines the site and routes; every route reads its own remote
`.json` or `.jsonl` page object through short-lived GitHub OIDC credentials.

## Functional Requirements

- **F1 — Thin manifest:** `SITE_CATALOG_URI` points to a private `site.json` containing only site metadata,
  route definitions, page-source URIs, and field projections. It must not contain page records.
- **F2 — One object per route:** each route has one unique `data.source` ending in `.json` or `.jsonl`.
  Adding or replacing page content must not require editing a shared dataset object.
- **F3 — Source preference:** `data.source` may reference an original HTTPS source or a private GCS object.
  `data.fallback`, when present, is a private GCS `.json` or `.jsonl` snapshot.
- **F4 — Format behavior:** JSON may be an array or an object selected through `view.collection`. JSONL is
  parsed as one JSON record per non-empty line and therefore resolves directly to an array.
- **F5 — Data-defined routes:** every route selects the `catalog` template. The root route `/` is required;
  non-root routes are generated by one catch-all route.
- **F6 — Schema projection:** `view` maps page-source fields to title, summary, tags, and URL. Compatible
  data and routes require no TypeScript change.
- **F7 — Dual format:** every page is emitted as accessible HTML and `text/markdown`. The Markdown graph
  must be reachable from `/index.md`.
- **F8 — Static deployment:** a push to `main` checks, builds, and publishes `dist/` to GitHub Pages.

## Security Requirements

- **S1 — No keys:** GitHub Actions uses direct Workload Identity Federation. Do not create or store a Google
  service-account key.
- **S2 — Restricted trust:** the provider condition restricts immutable GitHub owner ID, repository ID,
  `refs/heads/main`, and `.github/workflows/deploy.yml` on `main`.
- **S3 — Least privilege:** the repository principal receives only `roles/storage.objectViewer` on the
  manifest and page-object prefix, never project-wide storage access.
- **S4 — Private input, public output:** anonymous access to the manifest and every GCS page object returns
  403. Generated Pages output is public, so remote page data must contain publishable content, never secrets.
- **S5 — Safe connectors:** only `gs://` and `http(s)://` are supported. Filesystem paths and `file://` are
  rejected, and GitHub tokens are sent only to known GitHub hosts.
- **S6 — No credential artifacts:** generated `gha-creds-*.json` files and local `.env` files remain ignored.
- **S7 — Immutable Actions:** every external GitHub Action is pinned to a reviewed full-length commit SHA.

## Manifest Contract

The complete machine-readable contract is `catalog.schema.json`. A minimal `site.json` is:

```json
{
  "schemaVersion": 2,
  "site": {
    "title": "Example Catalog",
    "description": "A site built from private remote page objects."
  },
  "routes": [
    {
      "id": "home",
      "path": "/",
      "label": "Home",
      "template": "catalog",
      "data": {
        "source": "gs://BUCKET/PREFIX/pages/home.json"
      },
      "intro": {
        "eyebrow": "Seed",
        "title": "Remote resources",
        "summary": "One remote data object powers this page."
      },
      "view": {
        "collection": "items",
        "titleField": "name",
        "summaryField": "description",
        "tagsField": "tags",
        "urlField": "url"
      }
    },
    {
      "id": "events",
      "path": "/events/",
      "label": "Events",
      "template": "catalog",
      "data": {
        "source": "https://raw.githubusercontent.com/OWNER/SOURCE/main/events.jsonl",
        "fallback": "gs://BUCKET/PREFIX/pages/events.jsonl"
      },
      "intro": {
        "eyebrow": "JSONL",
        "title": "Events",
        "summary": "One record per line from the original source or private snapshot."
      },
      "view": {
        "titleField": "name",
        "summaryField": "description",
        "tagsField": "tags",
        "urlField": "url"
      }
    }
  ]
}
```

The corresponding `pages/home.json` may contain an object with a selected collection:

```json
{
  "items": [
    {
      "name": "Astro",
      "description": "The web framework used by this seed.",
      "tags": ["web", "static"],
      "url": "https://astro.build/"
    }
  ]
}
```

The corresponding `pages/events.jsonl` contains one page record per non-empty line and omits
`view.collection`:

```jsonl
{"name":"Launch","description":"Initial release","tags":["release"],"url":"https://example.com/launch"}
{"name":"Review","description":"Architecture review","tags":["architecture"],"url":"https://example.com/review"}
```

## Quality Requirements

- **Q1 — Minimal boundary:** transport and JSON/JSONL parsing exist only in `src/lib/connectors.ts`; route
  interpretation exists only in `src/lib/catalog.ts`; presentation exists under `src/components` and
  `src/layouts`.
- **Q2 — Failure over drift:** missing configuration, unsupported extensions, malformed JSONL records,
  duplicate page sources, unavailable sources without a fallback, or non-array projections fail the build.
- **Q3 — Reproducibility:** dependencies are locked and CI uses `npm ci`.
- **Q4 — Responsive baseline:** the single template works at desktop and mobile widths.

## Definition of Done

1. `npm ci`, `npm run check`, and `npm run build` pass against an authenticated private GCS manifest.
2. Validation covers at least one JSON route and one JSONL route, including a GCS fallback.
3. The build contains `/index.html`, `/index.md`, and both formats for every manifest route.
4. `npm run check:markdown:dist` reports no broken or unreachable Markdown pages.
5. GitHub Actions build and deploy jobs succeed using OIDC/WIF and repository variables only.
6. The deployed HTML and Markdown URLs return HTTP 200.
7. Anonymous access to the manifest and private page objects returns HTTP 403.
8. No governed data, downloaded manifest, page object, credential, or secret is committed.
