# Seed Requirements

## Objective

Provide the smallest practical Astro seed that builds a public GitHub Pages site from governed data kept
outside the repository. The production build reads a private catalog from Google Cloud Storage through
short-lived GitHub OIDC credentials.

## Functional Requirements

- **F1 — Remote catalog:** `SITE_CATALOG_URI` is the only catalog entrypoint. Production uses an exact
  private `gs://` object.
- **F2 — Source preference:** a dataset may declare `$source` as its original `https://` source and
  `$fallback` as a private GCS JSON snapshot. Inline datasets live inside the remote GCS catalog.
- **F3 — Data-defined routes:** every route is declared in the catalog and selects the `catalog` template.
  The root route `/` is required. Non-root routes are generated by one catch-all route.
- **F4 — Schema projection:** `view` maps source fields to the four presentation fields: title, summary,
  tags, and URL. Adding compatible data or routes requires no TypeScript change.
- **F5 — Dual format:** every page is emitted as accessible HTML and as `text/markdown`. The generated
  Markdown graph must be reachable from `/index.md`.
- **F6 — Static deployment:** a push to `main` checks, builds, and publishes `dist/` to GitHub Pages.

## Security Requirements

- **S1 — No keys:** GitHub Actions must use direct Workload Identity Federation. Do not create or store a
  Google service-account key.
- **S2 — Restricted trust:** the provider condition must restrict immutable GitHub owner ID, repository ID,
  `refs/heads/main`, and `.github/workflows/deploy.yml` on `main`.
- **S3 — Least privilege:** the repository principal receives only `roles/storage.objectViewer` on the
  catalog prefix or managed folder, never project-wide storage access.
- **S4 — Private input, public output:** anonymous access to the catalog must return 403. Generated Pages
  output is public, so the catalog must contain publishable content and never secrets.
- **S5 — Safe connectors:** only `gs://` and `http(s)://` are supported. Filesystem paths and `file://` are
  rejected, and GitHub tokens are sent only to known GitHub hosts.
- **S6 — No credential artifacts:** generated `gha-creds-*.json` files and local `.env` files remain ignored.
- **S7 — Immutable Actions:** every external GitHub Action is pinned to a reviewed full-length commit SHA.

## Catalog Contract

The complete machine-readable contract is `catalog.schema.json`. This is the smallest valid example:

```json
{
  "schemaVersion": 1,
  "site": {
    "title": "Example Catalog",
    "description": "A site built from private remote configuration."
  },
  "routes": [
    {
      "id": "home",
      "path": "/",
      "label": "Home",
      "template": "catalog",
      "dataset": "resources",
      "intro": {
        "eyebrow": "Seed",
        "title": "Remote resources",
        "summary": "Content resolved at build time from outside the repository."
      },
      "view": {
        "collection": "items",
        "titleField": "name",
        "summaryField": "description",
        "tagsField": "tags",
        "urlField": "url"
      }
    }
  ],
  "datasets": {
    "resources": {
      "items": [
        {
          "name": "Astro",
          "description": "The web framework used by this seed.",
          "tags": ["web", "static"],
          "url": "https://astro.build/"
        }
      ]
    }
  }
}
```

To prefer an original source with a GCS fallback, replace the inline dataset with:

```json
{
  "$source": "https://raw.githubusercontent.com/OWNER/REPOSITORY/main/resources.json",
  "$fallback": "gs://BUCKET/PREFIX/resources.json"
}
```

## Quality Requirements

- **Q1 — Minimal boundary:** transport code exists only in `src/lib/connectors.ts`; catalog interpretation
  exists only in `src/lib/catalog.ts`; presentation exists only under `src/components` and `src/layouts`.
- **Q2 — Failure over drift:** missing configuration, invalid routes, unavailable sources without a fallback,
  or non-array route collections must fail the build with a useful error.
- **Q3 — Reproducibility:** dependencies are locked and CI uses `npm ci`.
- **Q4 — Responsive baseline:** the single template must work at desktop and mobile widths.

## Definition of Done

1. `npm ci`, `npm run check`, and `npm run build` pass against an authenticated private GCS catalog.
2. The build contains `/index.html`, `/index.md`, and both formats for every remote route.
3. `npm run check:markdown:dist` reports no broken or unreachable Markdown pages.
4. The GitHub Actions build and deploy jobs succeed using OIDC/WIF and repository variables only.
5. The deployed HTML and Markdown URLs return HTTP 200.
6. Anonymous access to `SITE_CATALOG_URI` returns HTTP 403.
7. No governed data, downloaded catalog, credential, or secret is committed.
